## Design Philosophy

**Geometric Abstraction. Not Literal.**

Rauno's illustrations for Corelith follow a strict visual language:
- **Pure geometric shapes** — circles, rounded rectangles, arcs, grid lines
- **Limited palette** — coral (#F27B6A), violet (#7C5CFC), charcoal (#1A1A1A), warm white (#FAF9F7), cream (#F5F3EF)
- **No photorealism** — no photos, no 3D renders, no gradients on shapes
- **Purposeful composition** — every element supports the narrative of the page
- **Consistent stroke weight** — 1.5px for lines, matching Hugeicons icon system
- **Depth through overlap** — shapes overlap with 8-16px offsets, not drop shadows

**The Corelith Visual Signature:**
Our logomark is a coral circle. Every illustration should reference this:
- Circles appear as nodes, buttons, or focal points
- Rounded corners everywhere (radius = 8-12px, matching UI)
- Grid lines suggest structure/organization

## PART 3: PRODUCT UI MOCKUPS — Workflow Components (8 total)

These show interface components abstractly — not screenshots, but stylized representations of workflows. Used on product page and solution feature sections.

### Style Rules for UI Mockups:
- All corners: 8px radius (matching real UI)
- All strokes: 1px, #E5E5E5 for borders
- Background: white (#FFFFFF) or cream (#F5F3EF)
- Accent colors: coral for primary actions, violet for secondary
- Text represented by horizontal lines (not real text)
  - Title: 2px tall, 60-80px wide, charcoal
  - Body: 1px tall, 40-60px wide, charcoal-500
- Icons: small 16px circles or rectangles in accent colors
- Shadows: subtle, 0 2px 8px rgba(26,26,26,0.06)

- Max file size per SVG: 15KB
- All SVGs must have `viewBox` and no fixed width/height attributes
